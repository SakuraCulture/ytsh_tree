# 多仓一期技术文档

## 1. 目标与范围

### 1.1 业务目标
- 业务仓库独立建设，不复用 ERP 主数据。
- ERP 仅作为页面结构、交互模式、接口拆分方式的参考，不直接复用业务口径。
- 一期聚焦“主数据 + 采购闭环”，保证基础建档、采购、入库、库存查询可联调。

### 1.2 一期包含内容
- 仓库管理
- 供应商管理
- 仓库商品管理
- 库存查询
- 采购订单管理
- 库存流水接口预留

### 1.3 一期暂不包含
- 独立库存流水前端菜单页
- 人工库存调整
- 超出采购入库主链路的扩展仓储流程

## 3. 状态口径

### 3.1 仓库 / 供应商状态
业务后端口径：
- `1 = 正常`
- `0 = 停用`

说明：
- 该口径与项目已有 `COMMON_STATUS` 的 `0=开启 / 1=关闭` 语义冲突。
- 因此前端多仓页面未直接复用 `DICT_TYPE.COMMON_STATUS`，而是统一使用本地映射，避免展示与提交值错位。

### 3.2 采购订单状态
- `0 = 草稿`
- `1 = 待审核`
- `2 = 已审核`
- `3 = 已入库`
- `4 = 已完成`
- `5 = 已取消`

### 3.3 收货状态
- `1 = 待收货`
- `2 = 部分收货`
- `3 = 已收货`
- `4 = 有差异`

### 3.4 库存流水业务类型
一期采购入库使用：
- `70 = 采购入库`

## 4. 采购闭环说明

### 4.1 状态流转
- 草稿 → 提交 → 待审核
- 待审核 → 审核 → 已审核
- 已审核 → 确认入库 → 已入库
- 草稿 / 待审核 / 已审核 → 取消 → 已取消

### 4.2 后端约束
- 只有草稿态可编辑、删除、提交。
- 只有待审核态可审核。
- 只有已审核态可确认入库。
- 已入库、已完成、已取消不能再取消。

### 4.3 入库联动
确认入库后，后端会同步执行：
- 增加仓库库存数量与可用量
- 回写仓库商品“首次有库存日期 / 最近入库日期”
- 生成库存流水记录
- 更新采购单收货状态与入库数量

## 5. 前端实现

### 5.1 API 文件
位于：`ytsh-ui-vue3/src/api/business/warehouse/`

文件清单：
- `warehouse.ts`
- `supplier.ts`
- `product.ts`
- `stock.ts`
- `purchase.ts`

实现原则：
- 参数名与后端 Controller/VO 保持一致。
- 不沿用 ERP 旧命名口径。
- 导出统一走 `request.download`。
- 采购动作接口单独暴露为提交、审核、确认入库、取消方法。

### 5.2 页面文件
位于：`ytsh-ui-vue3/src/views/business/warehouse/`

#### 仓库
- `warehouse/index.vue`
- `warehouse/WarehouseForm.vue`

能力：
- 分页搜索
- 新增 / 编辑 / 删除
- 默认仓切换
- 导出

#### 供应商
- `supplier/index.vue`
- `supplier/SupplierForm.vue`

能力：
- 分页搜索
- 新增 / 编辑 / 删除
- 导出

#### 仓库商品
- `product/index.vue`
- `product/ProductForm.vue`

能力：
- 分页搜索
- 新增 / 编辑 / 删除
- SKU 选择弹窗
- 导出

实现约束：
- 删除时若后端返回“存在库存不能删除”，前端只提示，不做绕过处理。

#### 库存查询
- `stock/index.vue`

能力：
- 分页搜索
- 汇总卡片
- 导出

展示字段：
- 库存数量
- 可用量
- 在途量
- 冻结量
- 缺货时长

#### 采购订单
- `purchase/index.vue`
- `purchase/PurchaseForm.vue`

能力：
- 分页搜索
- 新增 / 详情 / 编辑 / 删除
- 提交 / 审核 / 确认入库 / 取消
- 主表 + 明细表单
- 导出

关键实现：
- 按仓库加载可选仓库商品，再结合 SKU simple-list 展示编码与名称。
- 切换仓库时清空明细已选 SKU，避免跨仓误选。
- 采购金额由采购单价 × 采购数量自动计算。
- 明细支持显示已入库数量、差异数、退货数。

## 6. 联调与验收建议

### 6.1 前端静态检查
在 `ytsh-ui-vue3` 目录执行：

```bash
pnpm ts:check
pnpm lint:eslint
```

### 6.2 页面联调顺序
1. 创建仓库并设置默认仓
2. 创建供应商
3. 创建仓库商品
4. 创建采购单并保存草稿
5. 提交采购单
6. 审核采购单
7. 确认入库
8. 检查库存页数量变化
9. 检查仓库商品首次有库存日期 / 最近入库日期
10. 如补做库存流水页，再核对库存流水记录

### 6.3 建议联调关注点
- simple-list 是否只返回正常状态数据
- 默认仓切换是否能正确互斥
- 仓库商品删除限制是否符合后端库存校验
- 采购按钮是否严格按状态显示
- 入库后库存统计卡片是否同步变化
