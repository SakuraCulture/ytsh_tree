# 测试环境绕过前端登录滑块设计

## 背景
当前前端登录页在开启验证码时，会在点击登录后弹出 `Verify` 组件进行滑块校验，校验成功后才调用登录接口。这会阻碍 agent / Playwright 的自动化测试，因为自动化流程难以稳定完成滑块交互。

本次目标不是关闭整套登录鉴权，而是仅在测试场景下绕过前端滑块校验，保留账号密码登录流程和后端登录接口不变。

## 现状
- 登录按钮入口在 `ytsh-ui-vue3/src/views/login/components/LoginForm.vue` 的 `getCode()`。
- 当前逻辑由 `VITE_APP_CAPTCHA_ENABLE` 控制：
  - 为 `false` 时，直接调用 `handleLogin({})`
  - 为 `true` 时，弹出 `Verify` 组件，校验成功后通过 `@success="handleLogin"` 继续登录
- 真正的登录请求仍然调用 `LoginApi.login()`，并在成功后写入 token、跳转路由。

## 目标
- 仅在显式测试开关开启时绕过前端滑块。
- 默认行为保持不变。
- 不修改后端接口协议，不引入隐式浏览器检测逻辑。
- 让 Playwright 或 agent 只需使用指定环境启动前端，即可正常填写账号密码并登录。

## 方案对比

### 方案 A：新增显式环境变量开关，前端跳过滑块（推荐）
新增类似 `VITE_DISABLE_LOGIN_CAPTCHA` 的环境变量，在登录组件内与 `VITE_APP_CAPTCHA_ENABLE` 组合判断。

优点：
- 改动范围最小，只影响登录页。
- 显式、可审计、可回退。
- 适合 Playwright，通过单独启动模式或 `.env` 配置控制。

缺点：
- 需要维护一个额外环境变量。

### 方案 B：自动识别自动化环境后跳过
根据 `navigator.webdriver`、URL 参数或其他运行时信号自动绕过。

优点：
- 测试启动后无需单独记环境变量。

缺点：
- 行为隐式，后续排查时不直观。
- 依赖运行环境特征，稳定性和可维护性较差。

### 方案 C：后端放松验证码/滑块校验
保留前端行为，但在测试环境放过后端相关校验。

优点：
- 可覆盖更完整的链路。

缺点：
- 影响面扩大到后端，不符合“仅关闭前端滑块阻碍”的最小目标。

## 选定方案
采用方案 A：新增显式环境变量开关，在前端登录页直接跳过滑块校验流程。

## 设计细节

### 1. 新增环境变量
在前端 Vite 环境中新增布尔字符串变量，例如：
- `VITE_DISABLE_LOGIN_CAPTCHA=false`

默认值为 `false`。仅测试环境把它设为 `true`。

建议放置方式：
- 本地/联调用 `.env.local` 或专门的测试环境文件控制。
- 若需要稳定供自动化使用，可新增专用 mode，例如 `env.e2e` 或 `test` 对应环境文件。

### 2. 登录组件判断逻辑
修改 `ytsh-ui-vue3/src/views/login/components/LoginForm.vue`：
- 读取 `VITE_DISABLE_LOGIN_CAPTCHA`
- 在 `getCode()` 中优先判断“是否禁用登录滑块”
- 若开关开启，则直接调用 `handleLogin({})`
- 若开关关闭，则继续沿用当前 `VITE_APP_CAPTCHA_ENABLE` 行为

新的优先级应为：
1. 若“禁用登录滑块”开启：直接登录
2. 否则若全局验证码关闭：直接登录
3. 否则：展示滑块验证

这样可以避免修改 `handleLogin()` 主体逻辑，保持 token 写入、跳转、记住我等行为不变。

### 3. Verify 组件展示逻辑
`Verify` 组件可以保留现状，不必删除。

推荐做法：
- 保持模板结构不变，减少 diff
- 仅通过 `getCode()` 的分支逻辑避免触发 `verify.value.show()`

这样即使验证码开关仍为 `true`，测试环境也不会弹出滑块。

### 4. 自动化测试使用方式
自动化测试启动前端时，使用带开关的环境配置即可。

示例目标效果：
- 正常开发：`VITE_DISABLE_LOGIN_CAPTCHA=false`
- 自动化测试：`VITE_DISABLE_LOGIN_CAPTCHA=true`

Playwright 脚本本身无需处理滑块，只需：
- 打开登录页
- 输入用户名密码
- 点击登录
- 等待跳转或首页加载

## 影响范围
- 直接影响文件：`ytsh-ui-vue3/src/views/login/components/LoginForm.vue`
- 可能影响文件：对应的 `.env.*` 环境文件，或 `package.json` 脚本（如果要新增专用启动命令）
- 不影响：
  - 后端登录接口
  - token 存储逻辑
  - 路由鉴权逻辑
  - 非测试环境的正常登录流程

## 风险与控制

### 风险 1：误把测试开关带到非测试环境
控制方式：
- 默认值保持关闭
- 只在专用测试环境文件中开启
- 变量命名明确表达“仅用于禁用登录滑块”

### 风险 2：后续误以为整套验证码都关闭了
控制方式：
- 明确这是“登录页前端滑块绕过”，不是后端鉴权放开
- 保持 `VITE_APP_CAPTCHA_ENABLE` 原有语义不变

## 验证方式
实现后至少验证以下场景：
1. 默认环境下，点击登录仍会触发原有滑块。
2. 测试环境下，点击登录不会弹出滑块，而是直接发起登录请求。
3. 测试环境下，登录成功后仍能正常写入 token 并跳转。
4. 用户名/密码为空时，表单校验仍然生效。

## 实施边界
本次只处理“前端登录页滑块绕过”。
不包含：
- 后端验证码校验改造
- 自动创建测试账号
- 绕过整个登录页或首页免登录
- Playwright 脚本编写与 CI 接入
